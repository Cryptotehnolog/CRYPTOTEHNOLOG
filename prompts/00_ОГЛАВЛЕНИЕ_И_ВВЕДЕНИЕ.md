## 🎯 ВВЕДЕНИЕ

### Цель проекта

Превратить торгового бота v4.3.1 в **институциональную торговую платформу** уровня small prop firm / crypto fund, способную:

- Управлять капиталом $100M+ без пропорционального роста операционного риска
- Работать неделями автономно без вмешательства человека
- Самовосстанавливаться после сбоев через многоуровневые режимы деградации
- Гарантировать целостность данных при любых инцидентах
- Обеспечивать полный аудит всех действий системы

### Ключевые отличия от v4.3.1

| Критерий                   | v4.3.1                        | CRYPTOTEHNOLOG                            |
| -------------------------- | ----------------------------- | ----------------------------------------- |
| Уровень автономности       | Требует постоянного присмотра | Автономная работа недели/месяцы           |
| Восстановление после сбоев | Ручное вмешательство          | Автоматическое самовосстановление         |
| Масштаб капитала           | До $1M                        | $100M+                                    |
| Аудит действий             | Базовое логирование           | Полный криптографический аудит            |
| Управление конфигурацией   | Файлы конфигурации            | Версионированная конфигурация с подписями |
| Режимы деградации          | 2-3 режима                    | Многоуровневая деградация                 |
| Стрессоустойчивость        | Базовая                       | Stress testing + Monte Carlo              |

## 🏗️ АРХИТЕКТУРА СИСТЕМЫ

### Multi-Language Architecture

```
CRYPTOTEHNOLOG Platform
│
├── Control Plane (Python)
│   ├── State Machine
│   ├── Config Manager
│   ├── Risk Engine (Python orchestration)
│   ├── Portfolio Governor
│   └── Kill Switch
│
├── Data Plane
│   ├── Event Bus (Rust) ← критическая производительность
│   ├── Risk Ledger (Rust) ← атомарность операций
│   ├── Audit Chain (Rust) ← криптография
│   ├── Market Data Layer (Python)
│   ├── Intelligence Layer (Python)
│   └── Strategy Layer (Python)
│
├── Execution Layer (Rust + Python)
│   ├── Order Execution Core (Rust) ← низкая латентность
│   ├── Smart Order Router (Python) ← логика маршрутизации
│   └── Exchange Adapters (Python) ← API интеграция
│
├── Observability (Python + Web)
│   ├── Metrics Collector (Python)
│   ├── Web Dashboard (React + TypeScript)
│   └── Grafana/Prometheus Integration
│
└── Storage
    ├── PostgreSQL (состояния, аудит)
    ├── Redis (кеш, state machine)
    └── TimescaleDB (временные ряды)
```

## 🔒 SECURITY PRINCIPLES

1. **Secrets Management**

   - Все секреты в HashiCorp Vault
   - Никаких hardcoded credentials
   - Ротация ключей каждые 90 дней

2. **Configuration Integrity**

   - Подписи конфигураций (GPG/RSA)
   - Версионный контроль конфигов
   - Approved hash verification

3. **Audit Trail**

   - Cryptographic hashing audit chain
   - Immutable logs
   - Tamper-evident records

4. **Network Security**
   - TLS для всех внешних соединений
   - API rate limiting
   - Exchange API key permissions (read/trade only, no withdrawal)

def calculate_r_units(equity: Decimal, risk_percent: Decimal) -> Decimal:
    """
    Calculate R-units based on equity and risk percentage.

    Args:
        equity: Current equity in USD
        risk_percent: Risk percentage (e.g., 0.01 for 1%)

    Returns:
        R-unit value in USD

    Raises:
        ValueError: If equity <= 0 or risk_percent <= 0
    """
    if equity <= 0:
        raise ValueError("Equity must be positive")
    if risk_percent <= 0:
        raise ValueError("Risk percent must be positive")

    return equity * risk_percent
