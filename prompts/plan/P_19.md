## 🧱 АРХИТЕКТУРНЫЕ ГРАНИЦЫ

`P_19` должна сохранить уже установленную ownership discipline.

### 1. Paper layer использует existing truths, а не подменяет их

Paper layer:

- потребляет existing typed truths;
- строит narrow rehearsal / simulated state;
- не переписывает задним числом ownership соседних runtime layers.

### 2. Paper не подменяет Execution и OMS

`Execution`:

- владеет `order-intent` truth;
- не становится paper submodule.

`OMS`:

- владеет `order-state / lifecycle` truth;
- не становится paper-owned registry.

Paper layer может читать downstream OMS truth,
но не получает ownership над execution decisions или order lifecycle.

### 3. Paper не подменяет Manager и Validation

`Manager`:

- владеет narrow workflow-coordination truth.

`Validation`:

- владеет narrow review / evaluation truth.

Paper layer может использовать эти truths как вход rehearsal contour,
но не получает ownership над:

- workflow coordination;
- review / evaluation semantics;
- broader comparison platform semantics.

### 4. Paper не поглощает backtest / dashboard

`backtest`:

- остаётся отдельным supporting / adjacent contour;
- не становится скрытой частью paper runtime.

`dashboard`:

- остаётся отдельным operator-facing contour;
- не становится обязательной частью `P_19`.

### 5. Future analytics / ops lines не должны маскироваться под Paper

Даже если future paper-supporting lines later могут использовать paper truth,
в рамках `P_19` она не должна автоматически превращаться в:

- analytics / reporting platform;
- backtesting engine;
- dashboard-led operator surface;
- notifications / approval workflow platform;
- liquidation / emergency-ops platform.

## 🔌 RUNTIME / BOUNDARY DISCIPLINE

Фаза должна сохранить установленную runtime discipline проекта:

- explicit runtime entrypoint;
- явный `start()` / `stop()`;
- отсутствие hidden bootstrap;
- query/state-first surface;
- operator-visible diagnostics;
- явная degraded/readiness truth.

Composition root не должен:

- собирать paper domain context;
- вручную моделировать paper semantics;
- брать на себя ownership соседних layer truths.

Bootstrap должен только:

- собрать paper runtime;
- wiring-ить upstream / adjacent typed truths;
- встроить diagnostics в общую runtime/health truth.
