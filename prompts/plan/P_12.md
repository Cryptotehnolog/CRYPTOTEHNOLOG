завершённых `Strategy Foundation`, `Execution Foundation` и
`Opportunity / Selection Foundation`.

Фактическая задача фазы:

- ввести orchestration/meta layer как самостоятельный production-compatible consumer contour;

Главный принцип фазы:

> `P_12` строит foundation для узкого meta/orchestration contour, а не широкую workflow, protection или order-management platform.

## 🧱 АРХИТЕКТУРНЫЕ ГРАНИЦЫ

### 1. Meta / orchestration layer является consumer-ом existing truths

Orchestration layer:

- потребляет уже существующие typed truths;
- не владеет `market_data`, `analysis`, `intelligence`, `signals`, `strategy`, `execution`, `opportunity` или `risk` truth;
- не подменяет strategy layer;
- не подменяет execution layer;
- не подменяет opportunity layer;
- не подменяет portfolio governance или protection line.

### 2. Bootstrap не владеет orchestration semantics

Composition root:

- wiring-ит existing runtime contours;
- передаёт orchestration layer только existing typed truths;
- не собирает orchestration contracts вместо orchestration layer;
- не реализует arbitration / conflict-resolution logic внутри bootstrap.

### 3. Event truth не подменяет state/query truth

Orchestration layer может публиковать typed events, но основная truth должна оставаться:

- query/state-first;
- operator-visible;
- deterministic.

### 4. Runtime-time truth живёт в runtime

Expiry / invalidation / degraded semantics определяются в runtime, а не маскируются под dataclass-level temporal truth.

### 5. Meta / orchestration layer не подменяет later control-plane lines

Первый orchestration contour не должен автоматически тянуть:

- full `StrategyManager`;
- broad workflow automation;
- notifications / alerting line;
- portfolio-level supervision;
- kill switch / emergency controls;
- OMS / developed order-management.

Foundation сначала фиксирует ownership, contracts и runtime boundary.

## 🧩 RUNTIME / BOUNDARY DISCIPLINE

`P_12` строится по тем же дисциплинам, что уже закреплены в `P_7-P_11`:

- explicit runtime entrypoint;
- no hidden bootstrap;
- no import-time subscriptions;
- diagnostics sink как operator-visible truth;
- narrow ingest boundary;
- честное разделение structural invariants и runtime-time semantics.

Итоговый boundary фазы должен быть таким:

- orchestration runtime получает existing typed truths через явный ingest path;
- orchestration layer сама внутри собирает meta/orchestration context;
- bootstrap only wires upstream truths into runtime;
- query/state surface позволяет честно наблюдать orchestration state без необходимости читать только events.
