## 📋 АРХИТЕКТУРНЫЕ ТРЕБОВАНИЯ (SOLID для SFT)

### Single Responsibility (S)

Каждый класс имеет одну ответственность:

| Класс | Ответственность |
|-------|----------------|
| `ConfigLoader` | Загрузка конфигурации из источника (файл, Vault) |
| `ConfigParser` | Парсинг YAML/JSON в Pydantic модели |
| `ConfigValidator` | Валидация схемы и бизнес-правил |
| `ConfigSigner` | GPG проверка подписей |
| `ConfigRepository` | Хранение версий в PostgreSQL |
| `ConfigWatcher` | Мониторинг изменений файлов |
| `ConfigPublisher` | Публикация событий в Event Bus |

### Open/Closed (O)

Расширение без изменения существующего кода:

```
Новые источники конфигурации добавляются через:
- Новый класс ConfigProvider (наследует ConfigProviderProtocol)
- НЕ требует изменения ConfigManager

Пример добавления etcd источника:
class EtcdConfigProvider(ConfigProviderProtocol):
    """Provider для etcd."""
    async def load(self) -> dict: ...
```

### Liskov Substitution (L)

Все реализации `ConfigProviderProtocol` взаимозаменяемы:

```python
class ConfigProvider(Protocol):
    """Протокол для источников конфигурации."""
    async def load(self) -> dict[str, Any]: ...
    async def reload(self) -> dict[str, Any]: ...

# Взаимозаменяемы:
file_provider = FileConfigProvider(path="config.yaml")
vault_provider = VaultConfigProvider(paths={...})
env_provider = EnvConfigProvider()
```

### Interface Segregation (I)

Узкие протоколы для разных задач:

```python
# Вместо одного толстого интерфейса:
class IConfigLoader(Protocol):
    """Загрузка конфигурации."""
    async def load(self, path: Path) -> bytes: ...

class IConfigParser(Protocol):
    """Парсинг конфигурации."""
    def parse(self, data: bytes) -> dict[str, Any]: ...

class IConfigValidator(Protocol):
    """Валидация конфигурации."""
    def validate(self, config: dict) -> SystemConfig: ...

class IConfigSigner(Protocol):
    """Проверка подписей."""
    async def verify(self, path: Path) -> bool: ...
```

### Dependency Inversion (D)

Зависимости от абстракций, не от конкретных реализаций:

```python
class ConfigManager:
    """Основной класс с DI."""
    
    def __init__(
        self,
        loader: IConfigLoader,
        parser: IConfigParser,
        validator: IConfigValidator,
        signer: IConfigSigner,
        repository: IConfigRepository,
        event_bus: "IEventBus",
    ) -> None:
        self._loader = loader
        self._parser = parser
        # ...
```

```python
from typing import Protocol, Any
from pathlib import Path
from decimal import Decimal

class IConfigLoader(Protocol):
    """Загрузка конфигурации из источника."""
    
    async def load(self, source: str) -> bytes:
        """Загрузить конфигурацию."""
        ...
    
    async def reload(self) -> bytes:
        """Перезагрузить конфигурацию."""
        ...

class IConfigParser(Protocol):
    """Парсинг конфигурации в словарь."""
    
    def parse(self, data: bytes) -> dict[str, Any]:
        """Распарсить данные в словарь."""
        ...

class IConfigValidator(Protocol):
    """Валидация конфигурации."""
    
    def validate(self, data: dict[str, Any]) -> "SystemConfig":
        """Валидировать и вернуть типизированную модель."""
        ...

class IConfigSigner(Protocol):
    """Проверка GPG подписей."""
    
    async def verify(self, path: Path) -> bool:
        """Проверить подпись файла."""
        ...

class IConfigRepository(Protocol):
    """Хранение версий конфигурации."""
    
    async def save_version(
        self,
        version: str,
        content_hash: str,
        config_yaml: str,
        loaded_by: str,
    ) -> None:
        """Сохранить версию конфигурации."""
        ...
    
    async def get_history(self, limit: int = 10) -> list[dict]:
        """Получить историю версий."""
        ...

class IConfigWatcher(Protocol):
    """Мониторинг изменений файлов."""
    
    async def watch(self, paths: list[Path]) -> None:
        """Начать мониторинг."""
        ...
    
    def on_change(self, callback: callable) -> None:
        """Установить callback на изменение."""
        ...
```

### 6. CONFIG MANAGER (Основной класс)
**Файл:** `src/cryptotechnolog/config/manager.py` (НОВЫЙ)

```python
class ConfigManager:
    """
    Менеджер конфигурации с поддержкой hot reload и GPG верификации.
    
    Особенности:
    - DI через конструктор
    - Plugable providers (файл, Vault, env)
    - Hot reload без рестарта
    - История версий в PostgreSQL
    """
    
    def __init__(
        self,
        loader: IConfigLoader,
        parser: IConfigParser,
        validator: IConfigValidator,
        signer: IConfigSigner,
        repository: IConfigRepository,
        event_bus: "EnhancedEventBus",
    ) -> None:
        self._loader = loader
        self._parser = parser
        self._validator = validator
        self._signer = signer
        self._repository = repository
        self._event_bus = event_bus
        self._current_config: SystemConfig | None = None
    
    async def load(self) -> SystemConfig:
        """Загрузить и валидировать конфигурацию."""
        ...
    
    async def reload(self) -> SystemConfig:
        """Перезагрузить конфигурацию (hot reload)."""
        ...
```

## ✅ КРИТЕРИИ ПРИЁМКИ

2. ConfigManager может работать с любым provider без изменения кода (O)

4. Зависимости внедряются через конструктор (D)
