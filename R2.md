## 🔴 Ошибка 1: Rust - Неразрешенный импорт `cryptotechnolog_audit_chain`

### Лог ошибки (Rust Tests)

```
error[E0432]: unresolved import `cryptotechnolog_audit_chain`
  --> rust_components/crates/risk-ledger/src/ledger.rs:21:5
   |
21 | use cryptotechnolog_audit_chain::AuditChain;
   |     ^^^^^^^^^^^^^^^^^^^^^^^^^^^ use of unresolved module or unlinked crate

error[E0433]: failed to resolve: use of unresolved module or unlinked crate `cryptotechnolog_audit_chain`
   --> rust_components/crates/risk-ledger/src/ledger.rs:266:14
    |
266 |     ) -> Vec<cryptotechnolog_audit_chain::AuditRecord> {
    |              ^^^^^^^^^^^^^^^^^^^^^^^^^^^ use of unresolved module or unlinked crate
```

### Корневая причина

В файле **`rust_components/crates/audit-chain/Cargo.toml`** **отсутствует секция `[lib]`**.

#### Сравнение конфигураций:

**✅ risk-ledger/Cargo.toml (корректно):**
```toml
[lib]
name = "cryptotechnolog_risk_ledger"
path = "src/lib.rs"
crate-type = ["lib", "cdylib"]
```

**❌ audit-chain/Cargo.toml (некорректно):**
```toml
# Секция [lib] ОТСУТСТВУЕТ!
```

### Почему это критично

1. **Workspace resolution:** Cargo собирает workspace на основе `members` из корневого `Cargo.toml`
2. **Library linking:** Для использования crate как зависимости (`--extern`), он должен быть явно объявлен как библиотека
3. **Dependency graph:** Без `[lib]` cargo не включает crate в граф зависимостей для линковки
4. **Build order:** Cargo не понимает, что `audit-chain` нужно собрать ПЕРЕД `risk-ledger`

### Как исправить

Добавьте в файл **`rust_components/crates/audit-chain/Cargo.toml`**:

```toml
[lib]
name = "cryptotechnolog_audit_chain"
path = "src/lib.rs"
crate-type = ["lib"]
```

Или, если нужна FFI-совместимость:

```toml
[lib]
name = "cryptotechnolog_audit_chain"
path = "src/lib.rs"
crate-type = ["lib", "cdylib", "staticlib"]
```

## 🟡 Ошибка 2: Python mypy - Двойное определение имен (`no-redef`)

### Лог ошибки (Python Tests 3.12)

```
src/cryptotechnolog/__init__.pyi:17: error: Name "config" already defined (possibly by an import)  [no-redef]
src/cryptotechnolog/__init__.pyi:18: error: Name "data" already defined (possibly by an import)  [no-redef]
src/cryptotechnolog/__init__.pyi:19: error: Name "rust_bridge" already defined (possibly by an import)  [no-redef]
src/cryptotechnolog/config/__init__.pyi:13: error: Name "logging" already defined (possibly by an import)  [no-redef]
src/cryptotechnolog/config/__init__.pyi:14: error: Name "settings" already defined (possibly by an import)  [no-redef]
src/cryptotechnolog/data/__init__.pyi:12: error: Name "frame" already defined (possibly by an import)  [no-redef]
```

### Корневая причина

В `.pyi` (type stub) файлах происходит **двойное определение** одних и тех же имён:

#### Проблемный код (src/cryptotechnolog/__init__.pyi):

```python
from __future__ import annotations
from typing import TYPE_CHECKING, Any

if TYPE_CHECKING:
    from cryptotechnolog import config  # ← Первое определение
    from cryptotechnolog import data    # ← Первое определение
    from cryptotechnolog import rust_bridge  # ← Первое определение

__version__: str

# Re-exports
config: Any      # ← Второе определение - КОНФЛИКТ!
data: Any        # ← Второе определение - КОНФЛИКТ!
rust_bridge: Any # ← Второе определение - КОНФЛИКТ!
```

### Почему mypy жалуется

1. **TYPE_CHECKING блок:** mypy анализирует код внутри `if TYPE_CHECKING:` и видит импорты
2. **Аннотации:** Затем mypy видит аннотации `name: Any`
3. **Redefinition:** mypy считает это переопределением одного и того же имени
4. **no-redef rule:** Правило `no-redef` (включено по умолчанию) запрещает переопределение

### Варианты исправления

#### Вариант 1: Удалить дублирующие импорты (РЕКОМЕНДУЕТСЯ)

**Файл: `src/cryptotechnolog/__init__.pyi`**

```python
# ==================== CRYPTOTEHNOLOG Package Type Stubs ====================
from __future__ import annotations
from typing import Any

# ==================== Version ====================
__version__: str

# ==================== Re-exports ====================
config: Any
data: Any
rust_bridge: Any
```

**Файл: `src/cryptotechnolog/config/__init__.pyi`**

```python
# ==================== CRYPTOTEHNOLOG Config Type Stubs ====================
from __future__ import annotations
from typing import Any

# Re-exports
settings: Any
logging: Any
```

**Файл: `src/cryptotechnolog/data/__init__.pyi`**

```python
# ==================== CRYPTOTEHNOLOG Data Type Stubs ====================
from __future__ import annotations
from typing import Any

# Re-exports
frame: Any
```

#### Вариант 2: Использовать псевдонимы (если нужны реальные типы)

```python
from __future__ import annotations
from typing import TYPE_CHECKING

if TYPE_CHECKING:
    from cryptotechnolog.config import settings as _settings
    from cryptotechnolog.config import logging as _logging
    
    settings: type[_settings]
    logging: type[_logging]
```

#### Вариант 3: Использовать `__all__`

```python
from __future__ import annotations
from typing import Any

__all__ = ["config", "data", "rust_bridge", "__version__"]

__version__: str
config: Any
data: Any
rust_bridge: Any
```
