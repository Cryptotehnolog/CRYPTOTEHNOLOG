# Мультиязычная архитектура Python + Rust

**Дата:** 2026-03-09  
**Статус:** Принято  

## Контекст
Фаза 0 определила фундаментальную архитектуру проекта CRYPTOTEHNOLOG как гибридную систему с двумя языками программирования.

- **Фаза:** 0 — Подготовка среды
- **Компонент:** Архитектура проекта
- **Класс стратегии:** Все классы (HFT/MFT/SFT)

### Почему потребовалось разделение:
1. **Python** — Control Plane, бизнес-логика, оркестрация
2. **Rust** — Data Plane, критические компоненты производительности

## Рассмотренные альтернативы
1. **Чистый Python** — проще разработка, но недостаточная производительность для HFT
2. **Чистый Rust** — максимальная производительность, но сложность разработки и долгий time-to-market
3. **Другие языки** (C++, Go) — не рассматривались из-за экосистемы

## Решение
Выбрана **гибридная архитектура** с чётким разделением:

- **Control Plane (Python):** State Machine, Risk Engine, Portfolio Governor, оркестрация
- **Data Plane (Rust):** Event Bus, Risk Ledger, Execution Core, Audit Chain
- **FFI слой:** crates/ffi для связи Python ↔ Rust

### Структура проекта:
```
CRYPTOTEHNOLOG/
├── src/cryptotechnolog/     # Python (Control Plane)
│   ├── core/               # State Machine, Health, Metrics
│   ├── config/             # Конфигурация
│   └── ...
├── crates/                 # Rust (Data Plane)
│   ├── eventbus/           # Высокопроизводительная шина
│   ├── risk-ledger/       # Двойная бухгалтерия рисков
│   ├── audit-chain/       # Криптографический аудит
│   ├── execution-core/    # Низкоуровневое исполнение
│   ├── ffi/               # Python bindings
│   └── common/            # Общие типы
```

## Последствия
### Плюсы:
- **Производительность:** Rust обеспечивает микросекундные задержки для HFT
- **Гибкость:** Python позволяет быстро прототипировать и добавлять новые стратегии
- **Разделение ответственности:** Data Plane не зависит от Control Plane
- **Экосистема:** лучшие библиотеки с обоих языков

### Минусы:
- **Сложность:** требуется понимание двух языков и FFI
- **Отладка:** усложнённая отладка через границу языков
- **CI/CD:** два набора инструментов (pytest + cargo)
