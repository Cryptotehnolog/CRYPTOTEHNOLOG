# Инварианты входа/выхода State Machine

**Дата:** 2026-03-09  
**Статус:** Принято  

## Контекст
State Machine должна гарантировать консистентность при переходах между состояниями. Без явных инвариантов возможны race conditions и некорректные переходы.
- Фаза 2 / Control Plane
- Класс: SFT

## Рассмотренные альтернативы
1. Оставить без изменений — риск race conditions
2. Добавить инварианты только для критических переходов — выбрано
3. Добавить полную формальную верификацию — избыточно для SFT

## Решение
Добавлены методы в `TradingStateMachine`:
- `get_entry_invariants(state)` — проверки при входе в состояние
- `get_exit_invariants(state)` — проверки при выходе из состояния

Пример инвариантов:
- Вход в RUNNING: all required components initialized
- Выход из RUNNING: no active orders или force=True

## Последствия
- **Плюсы:** Консистентность состояний, early error detection, документация контрактов
- **Минусы:** Небольшое увеличение CPU на проверки
