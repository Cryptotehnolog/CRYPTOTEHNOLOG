# FIX: Ошибка "unresolved import `cryptotechnolog_audit_chain`" в risk-ledger

**Дата:** 22 февраля 2026  
**Автор:** Senior Systems Architect (Grok)  
**Проект:** CRYPTOTEHNOLOG  
**Приоритет:** 🔴 ПРИОРИТЕТ 1 — Целостность данных и тестирование  

## ✅ Краткое резюме

**Проблема:**  
GitHub Actions падает на этапе проверки `risk-ledger` с ошибками:

```text
error[E0432]: unresolved import `cryptotechnolog_audit_chain`
error[E0433]: failed to resolve: use of unresolved module or unlinked crate `cryptotechnolog_audit_chain`

Причина:
В rust_components/crates/risk-ledger/Cargo.toml отсутствуют path-зависимости от внутренних кратов audit-chain и common.
Rust не может найти эти краты, хотя workspace их видит и собирает.

🔧 Полное исправление (5 минут)
Шаг 1: Обновить rust_components/crates/risk-ledger/Cargo.toml
Замените весь файл на следующее содержимое:

[package]
name = "cryptotechnolog-risk-ledger"
version = "0.1.0"
edition = "2021"
description = "Risk Ledger with AuditChain integration (Phase 1)"

[dependencies]
# === ВНУТРЕННИЕ КРАТЫ (ОБЯЗАТЕЛЬНО!) ===
cryptotechnolog-audit-chain = { path = "../audit-chain" }
cryptotechnolog-common      = { path = "../common" }

# === ВНЕШНИЕ ЗАВИСИМОСТИ ===
chrono = { version = "0.4", features = ["serde"] }
hex = "0.4"
serde = { version = "1.0", features = ["derive"] }
serde_json = "1.0"
sha2 = "0.10"
tokio = { version = "1", features = ["full"] }
tracing = "0.1"
uuid = { version = "1.0", features = ["v4", "serde"] }

# === PyO3 (для Python 3.12 интеграции) ===
pyo3 = { version = "0.23", features = [
    "extension-module",
    "abi3-py312",
    "multiple-pymethods",
    "anyhow"
] }

Шаг 2 (рекомендуется): Централизовать зависимости в корневом Cargo.toml
Добавьте в корневой Cargo.toml (после [workspace]):

[workspace.dependencies]
cryptotechnolog-audit-chain = { path = "rust_components/crates/audit-chain" }
cryptotechnolog-common      = { path = "rust_components/crates/common" }

Тогда в risk-ledger/Cargo.toml можно писать короче:

cryptotechnolog-audit-chain = { workspace = true }
cryptotechnolog-common      = { workspace = true }

Шаг 3: Проверить локально

cargo check -p cryptotechnolog-risk-ledger
# или весь workspace
cargo check --workspace

Должно быть чисто: Finished dev [unoptimized + debuginfo] target(s) in X.XXs
Шаг 4: Обновить GitHub Workflow (чтобы больше никогда не падало)
В .github/workflows/ci.yml (или любом другом) замените проверку Rust на:

- name: Check Rust workspace
  run: cargo check --workspace --all-targets
