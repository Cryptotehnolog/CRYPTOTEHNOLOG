# Исправление ошибки Rust - Workflow Run #22280078939

## ❌ Ошибка

```
error[E0432]: unresolved import `cryptotechnolog_audit_chain`
  --> rust_components/crates/risk-ledger/src/ledger.rs:21:5
```

## 🔍 Причина ошибки

Проблема в **некорректном разрешении зависимостей workspace**. 

В `risk-ledger/Cargo.toml` локальные зависимости указаны только с `path`:
```toml
cryptotechnolog-common = { path = "../common" }
cryptotechnolog-audit-chain = { path = "../audit-chain" }
```

Но cargo не может корректно разрешить эти зависимости при сборке, потому что:
1. Кэш cargo содержит устаревшие метаданные
2. Зависимости не объявлены в `[workspace.dependencies]` корневого Cargo.toml
3. Не используется `workspace = true` для локальных крейтов

---

## ✅ Решение

### Шаг 1: Обновить корневой Cargo.toml

Добавить локальные крейты в `[workspace.dependencies]`:

```toml
# ==================== CRYPTOTEHNOLOG Root Workspace ====================
[workspace]
resolver = "2"
members = [
    "rust_components/crates/common",
    "rust_components/crates/eventbus",
    "rust_components/crates/risk-ledger",
    "rust_components/crates/audit-chain",
    "rust_components/crates/execution-core",
    "rust_components/crates/ffi",
]

# Workspace package metadata
[workspace.package]
rust-version = "1.83"
edition = "2021"

# Workspace-level dependencies (shared across all crates)
[workspace.dependencies]
# ... существующие зависимости ...

# Локальные workspace members (для корректного разрешения зависимостей)
cryptotechnolog-common = { path = "rust_components/crates/common" }
cryptotechnolog-audit-chain = { path = "rust_components/crates/audit-chain" }
cryptotechnolog-eventbus = { path = "rust_components/crates/eventbus" }
cryptotechnolog-risk-ledger = { path = "rust_components/crates/risk-ledger" }
cryptotechnolog-execution-core = { path = "rust_components/crates/execution-core" }
cryptotechnolog-ffi = { path = "rust_components/crates/ffi" }
```

### Шаг 2: Обновить risk-ledger/Cargo.toml

Использовать `workspace = true` для локальных зависимостей:

```toml
[dependencies]
# Use workspace dependencies
tokio = { workspace = true }
serde = { workspace = true }
serde_json = { workspace = true }
chrono = { workspace = true }
tracing = { workspace = true }
uuid = { workspace = true }

# Cryptographic dependencies for Merkle tree
sha2 = "0.10"
hex = "0.4"

# Windows-specific dependencies for file sharing
[target.'cfg(windows)'.dependencies]
winapi = { version = "0.3", features = ["winnt"] }

# Local dependencies - workspace members (ИСПРАВЛЕНО)
cryptotechnolog-common = { workspace = true }
cryptotechnolog-audit-chain = { workspace = true }
```

### Шаг 3: Очистить кэш cargo (важно!)

Добавить в CI перед сборкой:

```yaml
- name: Clean cargo cache
  run: |
    cargo clean
    rm -rf ~/.cargo/registry/src/*/cryptotechnolog-* || true
    rm -rf ~/.cargo/registry/index/github.com-*/cryptotechnolog-* || true
    rm -rf target/debug/.fingerprint/cryptotechnolog-* || true
```

Или изменить ключ кэша в CI:

```yaml
- name: Cache cargo build
  uses: actions/cache@v4
  with:
    path: |
      target
      ~/.cargo/registry
      ~/.cargo/git
    key: ${{ runner.os }}-cargo-v2-${{ hashFiles('**/Cargo.lock') }}  # Изменено: v2
    restore-keys: |
      ${{ runner.os }}-cargo-v2-
```

### Шаг 4: Обновить CI workflow

Изменить порядок сборки:

```yaml
- name: Build workspace dependencies first
  run: |
    echo "=== Cleaning previous builds ==="
    cargo clean
    echo "=== Building dependencies in order ==="
    cargo build --package cryptotechnolog-common --lib
    cargo build --package cryptotechnolog-audit-chain --lib
    echo "=== Building risk-ledger ==="
    cargo build --package cryptotechnolog-risk-ledger --lib
```

## 📋 Альтернативное решение (если workspace = true не работает)

Если `workspace = true` вызывает проблемы, можно использовать `path` с явным указанием features:

```toml
# risk-ledger/Cargo.toml
[dependencies]
cryptotechnolog-common = { path = "../common", default-features = true }
cryptotechnolog-audit-chain = { path = "../audit-chain", default-features = true }
```

И убедиться, что все крейты имеют `default-features = true` или явно указанные features.

## ⚠️ Важные замечания

1. **Кэш cargo** - главная причина проблемы. После добавления `[lib]` секций нужно полностью очистить кэш.

2. **Порядок сборки** - cargo должен собирать зависимости в правильном порядке: common → audit-chain → risk-ledger

3. **Cargo.lock** - после изменений нужно перегенерировать Cargo.lock и закоммитить его

4. **CI кэш** - ключ кэша должен быть изменён, чтобы GitHub Actions не использовал старый кэш
